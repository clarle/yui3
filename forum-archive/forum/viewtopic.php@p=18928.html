Hi,

It might, yes, the 2in3 doesn't add a YAHOO variable to the global context, which is what you'd expect usually with YUI2, it just creates it in the Y sandbox. At the very least you'd need to add
Code:
YAHOO = Y.YUI2
within the sandbox script to make a global YAHOO object (note the lack of var).

The problem you're seeing is an ordering problem: watching the resources load in Chrome dev tools, it's getting button-min.js before yahoo-dom-event.js and element-js (and I don't know off the top of my head whatever other dependencies button has).

I'm guessing that because you're using 3.0, the loader doesn't 'know' about YUI2 stuff. In your YUI3 config and use statements, you don't seem to be loading yui2-yde, which will be needed before the yui2-button. You could try use('node', 'yui2-yde', 'yui2-button'...), but I don't know if load guarantees the load order. I'm afraid I don't know the internals well enough to know how and why and when things are loaded in these sorts of circumstances.
