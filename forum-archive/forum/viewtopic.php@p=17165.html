Posted: Fri Sep 17, 2010 10:03 am
+0-
I am imitating the YUI 2.8 datatable examples and trying to build a simple datatable-manipulation page that supports insert, deleting and inline editing. The question is that these examples only work at the client side. But only client side change doesn't make too much sense, you really need to make the data on the server to be synchronized as well.

Now, I could send the manipulation to the server using YAHOO.util.Connect.asyncRequest (GET or POST method), however, how could I hold the client side changes until I receive the response from the server, and then depending on the response to decide to accept the client side change or cancel the change?

If you have a working example (based on YUI 2.8 rather than other version), that will make my life easier, as I am kind of person who likes to learn from examples more than from just plain text.

I'm doing something very similar, here's a snip of my code using YUI 2.8.1 in case it helps. Keep in mind it is a partial example. it won't run exactly like this but it should give you the general idea

Two of my fields are start and end period. the rule is start <= end.
Code:
// ...snip of validator for end
editor: new YAHOO.widget.DropdownCellEditor({
    asyncSubmitter: function(callback, newValue) {

        var record = this.getRecord();
        var start = parseInt(record.getData('start'));
        if (start > newValue) {
            alert('Start must be <= End');
            callback(false, newValue); // tell the editor not to show the value
        } else {
            YAHOO.util.Connect.asyncRequest('POST', 'some-url', {
                success: function () {
                    callback(true, newValue); // notify the editor to show the new value
                },
                failure: function () {
                    callback(false, newValue); // tell the editor not to show the value
                }
            });
        }
},
//...snip

Yes, that is the idea of asynchronous operations, you can do other things while the operation carries on. You can use methods disable and undisable to block changes in the DataTable while such operations carry on. The asyncSubmitter does this automatically. This will not prevent actions on other components in the page.
