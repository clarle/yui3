I wanted to elaborate on Juan's comment about giving the base extension the name "panel".

Code:
// hack: call it the same so you get the same css class names
// this is good for demos and tests. probably not for real life
MyPanel.NAME = 'panel';


Let's say for example, you multiple panels and want to give them unique names. So you use:

Code:
MyPanel.NAME = 'creditCardPanel';


You then discover that .hide() no longer works!

The problem is, the panel widget's css rules are fixed, so you when the change the name, you have to add your own CSS style rules.

Default style rules that come with panel:

Code:
.yui3-panel-focused {
    display:"" /* this clears away the "none" and reverts to the default styling for the panel */
}

.yui3-panel-hidden {
    display:none; /* this class is applied to the panel as the last rule in the chain, hiding the panel */
}


So you need to add these styles to your style sheet when you change the name while extending the class.

Code:
.yui3-creditCardPanel-focused {
    display:""
}

.yui3-creditCardPanel-hidden {
    display:none;
}


Now, .hide() works and you have a uniquely named panel.

I've found as a YUI newb, that watching how the markup changes as I hide and unhide the panel helps me to understand what is going on.
