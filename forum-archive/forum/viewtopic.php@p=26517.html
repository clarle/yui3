Quote:
parseXMLData does not seem to work for XPath expressions on Safari. Is it a known problem?


I haven't heard that parsing XML is a particular problem in Safari. There are a few apps I have run that use XML data with DataSource / DataTable that run fine in Safari (and other webkit based browsers). What OS are you running Safari on?

Before getting into your "fix" that you pastied, maybe you could ensure that XML parsing is failing. I have a public demo of a local XML-based DataTable that I put together awhile back that works on FF, Chrome, Safari on Mac, Win and some Linux. IE is a bit of a problem, depending on how you "create" the XML from a string. A publicly accessible link to this demo is http://blunderalong.com/pub/dtb/dt_xml_xpath_demo.html.

Note that you don't need to necessarily override parseXMLData unless you are certain it is incorrect. The default method (as shown in the above example) works fine. We typically override the method to add new features / capability (I think what you intended with your pastie).

So in summary, I don't know if saying XML with XPath doesn't work in Safari is accurate. If you have a specific working example (demonstrating the failure) maybe the forum can help out more.

Regards,

Looked at your code, and I stuck your XML and schema / coldefs in an XML test I have but didn't have time to debug. I saw the DOMParser method you use and am not familiar with that nor if it works with Safari. I also noted on your code sample that you define a formatter on the last column and wasn't sure I saw that defined in the snippet you provided, not sure if that is an issue.

The XML DT sample I provided earlier works on Safari / FF / Chrome, etc... but conveniently doesn't work on IE. Funnily, your code works on IE but not Safari. Have you checked it on FF or Chrome?

The TYPE_XML setting of DataSource expects a DOM representation of the XML, as in a DOM fragment. That is why I created an empty DIV and stuck the XML in innerHTML. I assume maybe your DOMParser utility is similar. There may be an issue with Safari in that area.
